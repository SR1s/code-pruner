# 开发规范

## TDD 开发范式

本项目采用测试驱动开发（TDD）：

1. **先写测试，后写实现** - 任何功能必须有配套的测试用例
2. **红-绿-重构循环** - 先让测试失败（红），再写最少代码通过（绿），最后重构
3. **测试即文档** - 测试描述功能的行为和边界

## 测试运行

```bash
pytest -v              # 运行所有测试
pytest --cov         # 带覆盖率报告
```

## 项目结构

code-pruner/
├── code_pruner/      # 源码
